# HANDOFF — usdt-slope-strategies

> 最后更新 2026-07-03。写给第一次接手的人：这个库是干什么的、现在什么在跑、动手前必须知道的规矩、坑在哪。

## 0. 这个库是干什么的（先看这段）

一句话：**USDT 增发提速的时候买 BTC**。USDT 市值是链上法币进场的计量表，它的增速冲进历史高分位、且 BTC 自己也在涨的那几天（一年就出手几次），后面一到两周 BTC 平均有顺风。本库把这个想法做成了两个冻结策略（纯做多 / 多空），每天自动检查信号、推 Telegram、更新结果页面。策略参数是冻结的——每天只是延长评估窗口加新数据，**不是**每天重新调参。

这个信号验证过是真的，但量级要摆正：可执行 Sharpe 约 1.7-1.9（含成本），在场时间约 10%，超额主要来自"躲开大跌"而不是"抓到更多涨"。它不是每天都灵的预测器——全样本日线性相关只有 0.05 左右，优势集中在极少数事件日（详见 §4 研究结论）。

## 1. 现在什么在跑（实盘状态）

- **每日 cron**：OpenClaw 任务 `usdt-strategies-signal-check`（id 89fa8051），**root@225，每天 00:10 UTC**，做四件事：git pull → 刷数据（`scripts/live_signal.py` 内部调 `refresh_full_data`）→ 算两个策略当天动作（开/平/持有/空仓）→ 推 Telegram（660451611，经 225-hermes）+ commit/push 数据和页面。
  - 00:10 这个时间点有讲究：CoinGecko 当天 00:00 UTC 的快照点实测当天就能取到，取到就能立即按市价执行——见 §3 口径。
  - 提交/推送由环境开关控制（`MONITOR_COMMIT` / `MONITOR_PUSH`，默认开）。
  - 回滚备份：旧任务 `usdt-signal-check`、`hermes-v27-usdt-signal-check`（跑旧库）还在、处于 disabled。
  - 另有 systemd `usdt-slope-monitor.service` 是老的 Java 版监控，与本库无关，别动。
- **监控状态文件** `data/monitor_state.json` 进 git、由 cron 维护——**ubuntu 侧改代码前先 `git pull`**，不然会和 root 的自动提交冲突。
- **结果页面**：https://dddabtc.github.io/usdt-slope-strategies/ （GitHub Pages = main 分支 /docs 目录；index + 做多页 + 多空页 + 杠杆页 + 开放搜索页 + 机理页）。

## 2. 推荐配置（当前生效）

- 两个策略杠杆推荐都是 **3.0x**（四条件规则定的：插针 10% 零爆仓 / bootstrap 中位回撤 ≥-35% / bootstrap P5>0 / ≤半 Kelly；保守就用 2.0x）。
- 成本模型：手续费+滑点 5 bps/边；做多付 funding 3 bps/天，做空按 0 计；维持保证金 0.5%。
- 评估窗口：2021-09-27 → 最新（用户钦定；参数冻结所以延长窗口=加数据不是调参）。冻结窗（→2026-03-19）数字只留在页面说明里做对照。

## 3. 执行口径（本库最重要的一条规矩，错过两次）

**日线 bar = CoinGecko 每天 00:00 UTC 的快照，不是收盘聚合。USDT 增发链上实时可见，BTC 7×24 交易。所以信号当根 bar 立即成交（lag=0）是现实口径，不是偷看未来。**

- 把"+1 天成交"当唯一诚实口径是把快照误当股市收盘价——这是老研究库犯的错，被用户纠正过**两次**（2026-06-10 建库时、2026-07-03 读旧文档又被带偏一次）。别再犯。
- 24h 滞后只是最坏边界参考。alpha 集中在快照后头几小时：1x 含成本 0h +282% / 1h +270% / 6h +216% / 24h +80%。
- 引擎参数 `execution_delay_frac`：0=立即（头条口径），None+lag=1=最坏边界。
- **两口径都和老库冻结产物逐笔对过账**：`python3 tests_regression.py` 必须全绿（lag0=28 笔 +2326.7%，lag1=27 笔 +320.6%，2.5x 无成本）。改引擎前后都要跑。

## 4. 研究结论（都做完了，别重复劳动）

- **验证**（逐笔）：最坏口径 1x 无成本 +90.9% vs 持有 +65.7%（在场仅 10.5%），t=2.18，bootstrap 亏损率 1.3%，随机入场安慰剂胜 96.5%，10bps/边仍 +269.8%。
- **开放搜索**（experiments/open-search/，嵌套 walk-forward 8 折 + 12 条错位假序列零检验）：**F3 纯做多流动性事件族胜出**——8/8 折样本外全正、Sharpe 1.68、回撤 -8.1% vs 持有 -52.4%；比纯 BTC 动量消融多 82pp（流动性信号不是动量换皮）；空头不增益（几乎不触发、触发即亏）。V27 冻结配置是该族成员。
- **机理轮**（experiments/mechanism-round/）：事件响应即时+持续两周（day1 +0.69%、day14 +2.1%），剂量倒 U（93-97 分位是甜区）；但 5 个"更聪明"的规则变体全败给冻结规则——**规则形状已无剩余优化空间，剩余空间在数据**（交易所充值流、资金费、链上实时铸币流）。
- **杠杆扫描**（experiments/leverage-sweep/）：做多 3x +3299%/回撤-23.2%/Sharpe 1.89；全窗口下多空反超纯做多，与开放搜索"逐折重选时空头不增益"两个口径各自成立、页面有说明。
- 2026-07-03 会话补充：全日线性 IC≈0.05 与事件优势 +2%/7d（t≈2.4）是同一事实的两个投影——别拿"日线性相关低"来否定事件策略，也别拿事件策略给"每日预测模型"背书。

## 5. 数据层（2026-07-03 加了修订审计）

- 源 = CoinGecko 免费 API，每天取"当天第一个点"（=00:00 快照），近 7 天回填自愈；`data/*_full.csv` 里 7 天以前的行落盘后**永不重写**——这就是"实盘当时看到的值"的存档。
- **修订审计**（`src/data.py` `_audit_revisions`）：每次刷新把新下载的 365 天和已存定稿值逐日比对，有出入就追加 `data/revision_log.jsonl` + 打印警告，存量值不改。**首次核对（2026-07-03）：新管道写入的全部历史零修订**。哪天这文件冒出新行，说明 CoinGecko 在改历史，先查再信回测。
- **口径接缝（勿当 bug）**：2026-04-24 及以前的历史来自老研究库的取数代码（日内点取平均，效果≈标签晚一天），和新口径系统性不同（BTC 最大差 16%）；审计起点因此定在 2026-04-25。老时代内部自洽，回测没有前视，只是接缝处有一次性口径差。
- 新研究（btc-five-factor-research）用的 USDT 源是 DefiLlama，水平差 ~1B 属口径差，两边数字不能直接混着比。
