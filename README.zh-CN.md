# USDT 斜率策略 — 单纯做多 & 多空双向，杠杆可调

**Language / 语言：** [English](README.md) · [简体中文](README.zh-CN.md)

先说人话：这套策略的想法很简单——把 USDT 的总市值当成「进场资金的水位计」。
大量新钱换成 USDT 涌进来，水位快速上涨，币价大概率跟着涨，这时候买入 BTC，
等涨势减速就卖掉；反过来，水位在持续下降的大环境里突然加速往下掉（像 2022 年
那种资金集体撤退），就做空。原仓库
[usdt-slope-research](https://github.com/dddabtc/usdt-slope-research) 用 30 轮、
两万多次实验找出了这套规则；本仓库做了三件事：**把它重新验证了一遍（一笔不差），
补上真实交易成本，再加上做空能力和「最该用几倍杠杆」的答案。**

**交互结果页（与原仓库同款）：**

## 1. 原策略验证结论

动手前先用冻结数据、冻结参数、信号次日才成交的诚实口径把原版 V27 重新跑了一遍：

| 检验 | 结果 |
|------|------|
| 复现冻结基准 | ✅ 完全一致：27 笔交易，无成本 2.5x 下 +320.6%，Sharpe 0.97，最大回撤 −27.3% |
| 与原仓库产物逐笔对账 | ✅ 27 笔交易的日期、价格、收益分毫不差（`tests_regression.py`） |
| 跑赢一直拿着不动 | ✅ 1 倍无成本 +90.9%，同期 BTC 拿着不动 +65.7%，而且只有 10.5% 的时间在场内 |
| 入场时机是否真有价值 | ✅ 和 2000 组「同样规则但随机挑日子买」相比，胜过其中 **96.5%** |
| 统计显著性 | ✅ 平均每笔 +2.60%，t 值 2.18；重抽样后亏钱概率只有 1.3% |
| 扣手续费后还赚吗 | ✅ 每边 10 个基点成本下，2.5x 仍 +269.8%（免费时 +320.9%） |
| 爆仓风险 | ✅ 样本内 5 倍以内按日收盘价从未触及强平线 |

要紧的背景：原 README 顶部的 +2152%、Sharpe 5.57 是「信号当天就能成交」的旧口径，
原仓库自己也已把诚实口径定格在 +320.6%。本仓库只用诚实口径，并在此之上再扣成本。

## 2. 这次改了什么

1. **真实成本**：每边 5 个基点的手续费+滑点，多头每天 3 个基点的资金费，都按杠杆后
   的名义本金收取（均可配置）。原引擎是零成本交易。
2. **爆仓建模**：按日收盘价做逐仓保证金检查（维持保证金 0.5%），杠杆扫描里另设
   「日内插针 10%」压力列——日线数据看不到针，只有连插针都扛得住的杠杆才算安全。
3. **做空侧**（多空双向策略）：把做多的每条规则原样镜像，**不新增任何拟合参数**——
   斜率跌破布林下轨/收缩形态、BTC 三日下跌确认、斜率分位 ≤7%、谷底回升出场——
   另加一道闸门：只在 60 天 USDT 斜率为负（资金大环境在退潮）时才允许做空，这条
   regime 规则是原研究 V23 生产配置的镜像。实测不加闸门时 15 笔空单白忙一场还把
   回撤翻倍；加了之后剩 4 笔，整体为正。
4. **杠杆变成一个参数**，并给出推荐值（见下）。

## 3. 头条结果（诚实成交 + 成本）

冻结基准窗口 2021-09-27 → 2026-03-19，同期 BTC 拿着不动 +65.7%。
信号出现后第二天才成交，手续费与资金费已扣。

**单纯做多**（原版 V27 冻结参数）：

| 杠杆 | 总收益 | 年化 | Sharpe | 最大回撤 | 胜率 | 笔数 |
|------|--------|------|--------|----------|------|------|
| 1.0x | +76.8% | +13.6% | 0.83 | −12.5% | 59% | 27 |
| 2.0x | +183.8% | +26.2% | 0.85 | −24.2% | 59% | 27 |
| **3.0x（推荐）** | **+319.2%** | **+37.7%** | **0.88** | **−34.8%** | **59%** | **27** |

**多空双向**（V27 多头 + 镜像门控空头）：

| 杠杆 | 总收益 | 年化 | Sharpe | 最大回撤 | 胜率 | 笔数 |
|------|--------|------|--------|----------|------|------|
| 1.0x | +79.6% | +14.0% | 0.78 | −13.4% | 58% | 31（27多/4空） |
| 2.0x | +189.7% | +26.8% | 0.81 | −25.8% | 58% | 31 |
| **2.5x（推荐）** | **+254.9%** | **+32.7%** | **0.83** | **−31.6%** | **58%** | **31** |

空单天生就少——USDT 市值很少持续缩水（2022 年是唯一一段像样的撤退）。它的价值在于
保险：再来一次 2022 那种资金退潮，做多侧只能闲着，做空侧会自动接班（2022 年 6 月
投降那笔 1 倍下 +8.8%，2026 年 2 月 +2.0%）。

## 4. 最佳杠杆

方法：按每笔收益算对数增长（Kelly 视角），交易序列重抽样一万次，叠加按收盘价的
强平检查和 10% 插针压力，0.5–6.0 倍全网格重跑回测。详见
[杠杆扫描页](https://dddabtc.github.io/usdt-slope-strategies/visualization_leverage.html)。

| | 单纯做多 | 多空双向 |
|---|---|---|
| 增长最优（满 Kelly） | >6x（网格边缘；样本只有 27 笔，别信） | >6x（同样别信） |
| **推荐** | **3.0x** | **2.5x** |
| 保守（回撤 ≤ 约 25%） | 2.0x | 2.0x |

推荐规则（事先定好，不看结果挑）：在「插针压力下零爆仓、重抽样中位回撤不破 −35%、
重抽样 5% 分位收益仍为正、不超过增长最优的一半」四个条件下取最大的杠杆。样本内
卡住推荐值的是后两条：单纯做多在 3.25x 破回撤线、3.5x 起 5% 分位转亏；多空在
2.75x 起 5% 分位转亏。

为什么不用满 Kelly：30 来笔样本算出的 Kelly 基本被估计误差主导；日线看不到日内
插针；样本里最差一笔 −13%，3 倍下就是 −39%。打对折是教科书式的处理。

## 5. 怎么跑

```bash
pip install -r requirements.txt

python tests_regression.py        # 验证做多侧与原版 V27 完全一致
python run_backtest.py            # 两套策略，各用推荐杠杆
python run_backtest.py --leverage 2.0   # 想用几倍用几倍
python run_leverage_sweep.py      # 全网格 + 最佳杠杆页面
python scripts/refresh_data.py    # 拉最新 CoinGecko 数据并重新生成页面
```

所有策略参数（杠杆、费率、资金费、维持保证金、信号常数）都在
`src/engine.py` 的 `StrategyConfig` 里。

## 6. 提醒

- 27–31 笔平仓交易是小样本，上面每个数字的置信区间都很宽（扫描页的重抽样列
  给出了量化区间）。
- 头条指标冻结在 2026-03-19；之后的交易只作为 Post-benchmark 叠加展示，不进头条。
- 日线收盘价看不到日内插针；插针压力列是近似，不是保证。高杠杆实盘的强平可能
  比回测更糟。
- 资金费按固定值近似（多头 3 基点/天、空头 0），实际会波动甚至翻转。
- 不构成投资建议。
