# USDT 斜率策略 — 单纯做多 & 多空双向，杠杆可调

**Language / 语言：** [English](README.md) · [简体中文](README.zh-CN.md)

先说人话：这套策略把 USDT 的总市值当成「进场资金的水位计」。大量新钱换成 USDT
涌进来，水位快速上涨，币价大概率跟着涨，这时候买入 BTC，等涨势减速就卖；资金
大环境在退潮时水位突然加速下跌（2022 年那种集体撤退），就做空。关键的一点：
**USDT 增发是链上实时可见的，每天的数据点也只是 0 点整的一张快照，币市 24 小时
开门——所以信号一出几分钟内就能下单，不用等到第二天。** 等得越久肉越少：1 小时
内下单几乎不损失，等满 24 小时收益只剩三成。原仓库
[usdt-slope-research](https://github.com/dddabtc/usdt-slope-research) 用 30 轮、
两万多次实验找出了这套规则；本仓库把它一笔不差地验证了一遍，修正了执行时机的
口径，补上真实交易成本，再加上做空能力和「最该用几倍杠杆」的答案。

## 1. 原策略验证结论

动手前先验证。下面所有检验都故意用**最坏情况的执行假设**（信号出来后干等一天
才下单）——在这个吃亏口径下都站得住，下单越快只会更好：

| 检验 | 结果 |
|------|------|
| 复现两个冻结产物 | ✅ 完全一致：立即成交 28 笔 +2326.7%，等一天 27 笔 +320.6%（2.5x 无成本） |
| 与原仓库产物逐笔对账 | ✅ 两个口径全部交易的日期、价格、收益分毫不差（`tests_regression.py`） |
| 跑赢一直拿着不动 | ✅ 最坏口径 1 倍 +90.9%，同期 BTC 拿着不动 +65.7%，且只有 10.5% 的时间在场内 |
| 入场时机真有价值 | ✅ 和 2000 组「同样卖法、随机挑日子买」相比，胜过其中 **96.5%** |
| 统计显著性 | ✅ 平均每笔 +2.60%，t 值 2.18；重抽样后亏钱概率只有 1.3% |
| 扣手续费后还赚吗 | ✅ 每边 10 个基点成本下最坏口径仍 +269.8% |
| 爆仓风险 | ✅ 样本内 5 倍以内按日收盘价从未触及强平线 |

## 2. 执行时机——为什么"立即下单"才是对的口径

数据里每天那个点是 **0 点整（UTC）的快照**，不是"当天结束才算出来的汇总"：
标着 6 月 9 日的那行，6 月 9 日 0 点 0 分 1 秒就全部可知。而真正的底层事件——
USDT 增发——链上一确认全网都看得见，比快照还快；BTC 又是 7×24 交易，没有
"等开盘"。所以 0 点快照算出的信号，几分钟内就可成交。

原仓库把"+1 天"当成唯一诚实口径，那其实是**最坏边界**而不是默认值——它假设
操作者看见资金涌入后还干等 24 小时。本仓库的成交模型是「信号后延迟多久成交」
（`execution_delay_frac`），各反应速度下的收益衰减（单纯做多 1 倍、含成本）：

| 反应时间 | 0 小时（头条口径） | 1h | 3h | 6h | 12h | 24h（最坏边界） |
|---|---|---|---|---|---|---|
| 总收益 | +282.1% | +270.3% | +247.7% | +216.4% | +162.2% | +80.4% |

一小时内下单保住约 96%；肉集中在快照后的头几个小时——资金流入信号本来就该
长这样。哪怕用 6 小时一班的定时任务也还有约 77%。两个边界都由
`tests_regression.py` 与原仓库冻结产物逐笔对账锁定。

## 3. 这次改了什么

1. **执行模型**：成交延迟连续可调，头条=立即成交；24 小时最坏边界保留并有测试。
2. **真实成本**：每边 5 个基点手续费+滑点，多头每天 3 个基点资金费，按杠杆后
   名义本金收取（可配置）。原引擎是零成本交易。
3. **爆仓建模**：按日收盘价做逐仓保证金检查（维持保证金 0.5%），另设
   「日内插针 10%」压力列。
4. **做空侧**（多空双向）：做多规则原样镜像，**不新增任何拟合参数**——跌破布林
   下轨/收缩形态、BTC 三日下跌确认、斜率分位 ≤7%、谷底回升出场——再加一道
   「60 天 USDT 斜率为负才许做空」的闸门（原研究 V23 regime 规则的镜像）。
   不加闸门，空单会在牛市的 USDT 小幅回落时乱开，白忙一场还把回撤翻倍。
5. **杠杆变成一个参数**，并给出推荐值（见下）。

## 4. 头条结果（立即成交 + 成本）

评估窗口 2021-09-27 → 最新已结算日（cron 每天自动延伸；参数早已冻结，窗口
延长只是加数据不是调参）。截至 2026-06-10：同期 BTC 拿着不动 +46.2%。
手续费与资金费已扣，Sharpe 与最大回撤按日线计。

注意空头的口径差异：冻结参数跑全窗口时空头簿是净正贡献（2026 年收缩段的
那几笔），而逐折重选参数的开放搜索里空头不增益——两个说法在各自协议下都成立。

空单天生就少——USDT 市值很少持续缩水（2022 年是唯一一段像样的撤退）。它的价值
是保险：2022 年 6 月投降那笔空单 3 倍下 +20.9%，2026 年 2 月 +1.7%，而那些时段
做多侧只能闲着。

## 4b. 开放式验证——这真是最佳策略吗？

不预设答案，把策略空间重新摊开比一遍（`experiments/open-search/`）：5 个事先声明
的策略族、约 160 个候选，**嵌套 walk-forward**（参数只在滚动两年训练窗里选，选完
在紧接着的半年未见窗口只评一次，8 个窗口串联），再加**代理零检验**——把一模一样
的搜索在 12 条假 USDT 序列上重跑，量出"纯靠挖能挖出多少"。

| 策略族 | 未见窗串联收益 | Sharpe | 最大回撤 | 笔数 |
|---|---|---|---|---|
| 一直拿着 BTC | **+189.1%** | 0.84 | −52.4% | — |
| **USDT 做多事件族（胜出）** | +174.8% | **1.68** | **−8.1%** | 32 |
| 多空镜像 | +135.3% | 1.44 | −8.1% | 28 |
| 流动性 regime 切换 | +110.9% | 0.83 | −46.5% | 5 |
| 纯 BTC 动量（消融） | +92.5% | 0.85 | −22.8% | 56 |

结论说人话：**最好的策略就是"盯着 USDT 的钱什么时候涌进来、只做多"这一族。**
它是唯一八个未见窗口全部为正的（包括 2025-26 这轮拿着不动巨亏 −52% 的熊市段），
Sharpe 是所有对手的两倍，回撤只有拿着不动的六分之一，收益却拿到它的九成多；比
不用 USDT 的纯价格动量多赚 82 个百分点（流动性信号不是动量换皮）；12 条假序列
全被真实成绩甩开（约 3 倍于最好的假成绩；p=1/13≈0.077，12 条样本能给的下限）。
做空在诚实选择下没有净贡献。本仓库部署的 V27 冻结配置就是这一族的成员（USDT
斜率事件 + 一年分位前 7-10% + BTC 三日确认 + 动能衰减出场）。

## 4c. 机理研究——规则本身还有没有可解释的优化空间？

先说人话：把"USDT 增发如何变成 BTC 涨幅"这条因果链拆开量了一遍，再按机理提了
五种讲得清道理的规则改法逐一对决（`experiments/mechanism-round/`），结论是
**一个都打不过现行规则**：按市值归一化斜率（−130pp）、要求连续两天确认（−93pp）、
反着买"还没涨的"（−127pp）、改成价格走弱才出场（−72pp）、掐掉最极端前 3% 的
事件（事后假设，−97pp）。原因机理研究自己给出了：增发后的反应**既即时又持久**
（第 1 天就有超额 +0.7%，之后持续漂移约两周，第 4 天附近还有一波部署回声）；
剂量效应是倒 U 形（93-97 分位是甜区）；持仓收益到第 14 天还在累积——这个形状
恰好就是"立即进场+流量衰减出场"在收割的东西。剩下的可解释空间在**数据**上
（交易所充值流、资金费吸收、链上实时铸币流），不在规则几何上。

## 5. 最佳杠杆

方法：按每笔收益算对数增长（Kelly 视角），交易序列重抽样一万次，叠加按收盘价
的强平检查和 10% 插针压力，0.5–6.0 倍全网格重跑回测。详见
[杠杆扫描页](https://dddabtc.github.io/usdt-slope-strategies/visualization_leverage.html)。

| | 单纯做多 | 多空双向 |
|---|---|---|
| 增长最优（满 Kelly） | ≥6x（网格边缘；30 笔小样本，别信） | ≥6x（同样别信） |
| **推荐** | **3.0x** | **3.0x** |
| 保守（回撤 ≤ 约 20%） | 2.0x | 2.0x |
| 按 24h 最坏边界算 | 3.0x | 2.5x |

推荐规则（事先定好，不看结果挑）：在「插针压力下零爆仓、重抽样中位回撤不破
−35%、重抽样 5% 分位收益仍为正、不超过增长最优的一半」四个条件下取最大杠杆。
立即成交口径下样本内数据好到只有"不超过半 Kelly"这条在起作用——这恰恰是它
存在的意义：30 来笔交易加日线数据，真正保护你的是对估计误差的纪律，不是回测
曲线。

## 6. 怎么跑

```bash
pip install -r requirements.txt

python tests_regression.py        # 两个执行口径 == 原版 V27 冻结产物
python tests_verification.py      # 安慰剂/显著性/成本检验
python run_backtest.py            # 两套策略，推荐杠杆
python run_backtest.py --leverage 2.0   # 想用几倍用几倍
python run_leverage_sweep.py      # 全网格 + 最佳杠杆页面
python scripts/refresh_data.py    # 拉最新数据并重新生成页面
```

所有参数（杠杆、成交延迟、费率、资金费、维持保证金、信号常数）都在
`src/engine.py` 的 `StrategyConfig` 里。

## 7. 提醒

- 头条口径假设你在 0 点快照后几分钟内下单（盯链上增发或一个勤快的轮询脚本）。
  下单慢的代价见第 2 节衰减表；24 小时边界是地板。
- 30 来笔平仓交易是小样本，每个数字的置信区间都宽（扫描页的重抽样列有量化）。
- 评估窗口每天自动延伸到最新已结算日；冻结窗（→2026-03-19）数字保留在页面说明里作对照。
- 日线收盘价看不到日内插针；插针压力列是近似。高杠杆实盘的强平可能比回测更糟。
- 资金费按固定值近似（多头 3 基点/天、空头 0），实际会波动甚至翻转。
- 不构成投资建议。
